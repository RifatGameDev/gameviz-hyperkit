# Template Media Guide

This guide explains how HyperKit template screenshots, GIFs, thumbnails, and demo media should be organized.

HyperKit templates should eventually include visual media so new users can quickly understand what each template looks like before creating a project.

---

## Media Folder Structure

Template media should be stored inside:

`docs/media/templates/`

Each template has its own folder:

- `docs/media/templates/tap_counter/`
- `docs/media/templates/flappy_mini/`
- `docs/media/templates/swipe_runner/`
- `docs/media/templates/puzzle_game/`
- `docs/media/templates/quiz_game/`
- `docs/media/templates/simple_physics/`

---

## Recommended Media Files

Each template folder may include:

- `screenshot.png` for a still preview
- `demo.gif` for short gameplay preview
- `thumbnail.png` for documentation cards or future website use

Example:

`docs/media/templates/tap_counter/screenshot.png`

`docs/media/templates/tap_counter/demo.gif`

`docs/media/templates/tap_counter/thumbnail.png`

---

## Screenshot Guidelines

Recommended screenshot rules:

- show the main game screen
- use a clean default template state
- avoid personal/local file paths
- keep the image simple and readable
- prefer PNG for screenshots
- prefer GIF for short gameplay clips

---

## Demo GIF Guidelines

Recommended GIF rules:

- keep the GIF short
- show the main gameplay loop
- avoid very large file sizes
- show player interaction clearly
- use one GIF per template when possible

---

## Current Status

The media folders are currently placeholders.

Actual screenshots and GIF files will be added in a later phase.
